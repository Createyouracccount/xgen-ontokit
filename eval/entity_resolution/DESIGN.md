# ER(개체정규화) 축 심판루프 — 설계 노트

계층(Wikidata P279, 89/100)·관계(KLUE-RE, 90/100)에 이은 3축 로드맵의 마지막.
같은 프로토콜: 외부 gold → 결정적 개선 → 심판 88게이트 → 본체 배선.

## 과제 정의

**개체정규화(Entity Resolution) = 같은 실세계 개체를 가리키는 서로 다른 표기를 하나로.**
온톨로지에서 동의어가 별개 URI 로 쪼개지면 그래프가 파편화된다("이커머스"와 "전자상거래"가
따로 노드).

## 현 본체 baseline = 형태소 정규화 (DeterministicDedup)

`src/ontokit/dedup/deterministic.py` — Kiwi 명사키로 병합. **표면 변이만 잡는다.**

| 쌍 | 현 본체 | 유형 |
|---|---|---|
| 삼성전자 / 삼성 전자 | ✅ 병합 | 표면 변이(공백) |
| 코로나19 / 코로나-19 | ✅ 병합 | 표면 변이(하이픈) |
| 전자상거래 / 이커머스 | ❌ 별개 | **의미 변이** |
| COVID-19 / 코로나19 | ❌ 별개 | **의미 변이(언어)** |
| 인공지능 / AI | ❌ 별개 | **의미 변이(약어)** |

→ 계층 접미공유·관계 조사SVO 와 같은 구조적 한계("표면 문자열 ≠ 의미"). ER 축의
**개선 = 표면 다른 의미 동의어를 외부 사전으로 잡기**. 이게 순가치의 원천.

## 외부 gold (확정: 위키피디아 redirect)

한국어 위키피디아 redirect(CC BY-SA), positives 431(surface 23/semantic 408) + 하드
네거티브 82. **자체 합성 GT 금지**(계층 합성 0.96→외부 0.33 붕괴).

## 채널 결정 (심판 66/100 판정 반영, 2026-07-14)

1차 시도(Wikidata QID 조회)는 순가치 9.6%로 실패 → 개발자가 "텍스트ER 틀림"이라 자체
진단했으나 **심판이 반증**(날것 klue-roberta 임베딩이 residual 62%@P0.90 회수). 결론:
**주력 채널 = 로컬 임베딩 코사인 유사도**. 형태소 정규화가 원리적 불가한 의미변이
(전자상거래~이커머스 0.902)를 분포 신호로 잡음. 정본 docs/ontokit_ER_심판_텍스트접근_
채널교체_2026_07_14.md.

⚠️ **누수 정리**: 임베딩(klue-roberta)은 위키 redirect 를 참조하지 않는 독립 신호
(일반 코퍼스 사전학습) → **임베딩 채널 + 위키 gold = 누수 없음.** 심판이 경고한 누수는
"Wikidata 를 KB 정규화 자원으로 쓸 때"(WD altLabel⊂위키 redirect 자매복사)에 한함.
따라서 임베딩 주력 + 위키 gold 평가는 정당. WD 는 정밀도 보조로만(누수 주의).

## 평가 지표 (확정)

동의어 쌍 판정 = 이진 분류. gold 동의어쌍(양성) + 하드 네거티브(비동의어쌍).
- **Precision/Recall/F1**: 시스템이 "동의어"라 한 것 중 맞은 비율 / gold 중 잡은 비율
- 하드 네거티브 필수: 표면 유사하나 다른 개체(예: "사과(과일)" vs "사과(사죄)"),
  형태소키 충돌쌍. 안 그러면 "전부 동의어" 가 고득점(허수아비).
- 순가치 = 형태소 baseline 이 이미 잡는 표면변이를 **제외**한, 의미변이에서의 순증.
  (계층의 "순가치 분해"와 동일 — baseline 이 잡는 건 크레딧 없음)

## 누수 차단

추출 소스(정규화 대상 표기) ⊥ gold 소스(동의어 정답). 계층에서 Wikidata desc로
Wikidata edge 예측한 순환누수 전례 → 사전을 gold 로 쓰면서 동시에 정규화 자원으로
쓰면 순환. 평가는 held-out 사전 엔트리로.

## LLM-free 불변

정규화 = 외부 사전 조회(결정적) + 형태소 키. LLM API 0회. 임베딩(KURE 등) 사용은
경계선 — 로컬 추론이라 불변식과 양립하나 "결정적" 정의를 사용자와 확인 필요(로드맵 §4.2).
